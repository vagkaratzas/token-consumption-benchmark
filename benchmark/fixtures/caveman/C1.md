taskflow = small task-management app. Strict layered architecture: api/cli -> services -> repositories -> db.

- Models (taskflow/models): dataclasses (User, Task, Project) subclass BaseModel. BaseModel gives id/created_at/updated_at plus to_dict() and touch().
- DB (taskflow/db.py): framework-free in-memory Database. insert/get/list/update/delete over per-table dicts. Single persistence leaf.
- Repositories (taskflow/repositories): BaseRepository = generic CRUD, maps models to rows in one table. UserRepository/TaskRepository/ProjectRepository add domain queries (e.g. get_by_email, list_by_project).
- Services (taskflow/services): business logic. AuthService hashes passwords + authenticates. TaskService creates/assigns/completes tasks, calls NotificationService. ProjectService manages membership.
- API (taskflow/api): deps.build_services wires one Database into repositories + services (Services container). app.App = tiny (method, path) -> handler dispatcher. routes_* register handlers calling services.
- CLI (taskflow/cli.py): argparse front-end. Shares same services.

Dependency wiring centralized in build_services, so api + cli share one object graph. Each request flows down layers, back up with results.
